# Codex Complex Task Card + Three-Stage Acceptance

Use this template before executing any complex/multi-step task.

## Task Card (required, all 5 fields)

- Goal:
- Constraints:
- Non-goals:
- Done criteria:
- Verification commands:

If any field is empty, do not start implementation.

## Three-stage acceptance (required)

### 1) Plan check

- Approach is decision-complete.
- Boundaries and tradeoffs are explicit.
- Done criteria is testable.

### 2) Execution check

- Implemented scope matches plan.
- Non-goals were not accidentally implemented.
- Changed files are intentional and explainable.

### 3) Final check

- Verification commands actually ran.
- Output/evidence is captured.
- Completion statement references evidence.

## Front-end extra gate

For front-end work, completion requires both:

- UI checklist pass
- At least one real browser/page verification pass
